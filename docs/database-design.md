# Database Design — AbhiIterates.OS

> This document will be populated on Day 3 of Week 1.
> 
> Database design is intentionally deferred until Day 3 because:
> - Day 2 (User Journey & Information Architecture) will reveal all data requirements.
> - Designing the schema before understanding all entities leads to premature decisions.
> - A proper ER diagram requires knowledge of all relationships, which emerges from Day 2.
>
> **Do not create any database tables until Day 3.**

---

## Planned Domains

The following domains will be modeled on Day 3:

- **Identity:** users, roles, refresh_tokens, oauth_providers
- **Library:** subjects, resources, resource_access
- **Marketplace:** listings, purchases, payment_transactions
- **PDF Workspace:** highlights, bookmarks, annotations
- **AI:** ai_conversations, ai_messages, vector_embeddings
- **Notifications:** notifications, notification_preferences
